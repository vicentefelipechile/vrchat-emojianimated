# AGENTS.md — vrchat-emojianimate

> Reference guide for AI agents working on this codebase. Read this file before touching any code.

---

## 1. Project Identity

**vrchat-emojianimate** is a **portable, no-install desktop tool** that converts GIF/video files into **1024×1024 VRChat-compatible sprite sheets** (PNG, 8×8 grid, up to 64 frames, 128×128 px per cell).

- **Target audience:** VRChat avatar/world creators.
- **Output:** `<Name>_<N>frames_<FPS>fps.png` — a single flat PNG, no metadata embedded.
- **Distribution:** Single raw `.exe` (downloads FFmpeg on first launch).
- **Version:** 0.1.0

## 2. Technology Stack

| Layer | Technology | Notes |
|---|---|---|
| Desktop shell | **Tauri v2** | Rust backend, WebView2 renderer |
| Frontend | **Vanilla TypeScript** via Vite 6 | No React, no Vue, no framework |
| Frontend model | **Manual view router** | Class-based views, no virtual DOM |
| Video processing | **FFmpeg (downloaded)** | Downloaded to AppData on first launch |
| Media analysis | **FFprobe (downloaded)** | Downloaded to AppData on first launch |
| Image assembly | **Rust `image` crate 0.25** | PNG feature only; runs in-process |
| IPC | `tauri::command` + `invoke()` | All commands typed in `src/ipc.ts` |
| Build target | **Windows x64 primary** | macOS/Linux secondary |

**Hard rules on the frontend:**
- No TailwindCSS, no CSS-in-JS.
- No React/Vue/Angular/Svelte.
- No `border-radius > 4px` on controls, no gradients, no `box-shadow` everywhere.
- Font: **IBM Plex Sans** (UI) + **IBM Plex Mono** (filenames, values, code). Never Inter/Roboto/Open Sans.

## 6. IPC Contract (Frontend ↔ Backend)

All calls go through `src/ipc.ts`. Never call `invoke()` directly from views.
The Tauri side registers commands in `src-tauri/src/lib.rs`.

### `analyze_media`
```typescript
// Frontend call
analyzeMedia(path: string): Promise<MediaInfo>

// Rust: src-tauri/src/commands/analyze.rs
// Runs: ffprobe -v quiet -print_format json -show_streams <path>
interface MediaInfo {
  width:         number;
  height:        number;
  fps:           number;        // parsed from rational "30000/1001"
  total_frames:  number;        // -1 if unknown (estimated from duration * fps)
  duration_secs: number;
  format:        MediaFormat;   // enum: 'gif' | 'video'
}
```

### `extract_preview`
```typescript
// Frontend call
extractPreview(inputPath: string): Promise<string>  // returns base64 PNG

// Rust: src-tauri/src/commands/extract.rs → extract_preview()
// Runs: ffmpeg -y -i <input> -frames:v 1 -q:v 2 <tmp/preview.png>
// Returns: base64-encoded PNG string (no external crate — custom std encoder)
// Stored in: state.previewBase64 (cached for FitView)
```

### `extract_frames`
```typescript
// Frontend call
extractFrames(options: ExtractOptions): Promise<ExtractResult>

// IMPORTANT: invoke wraps in { options } → Rust receives ExtractOptions directly
// invoke<ExtractResult>('extract_frames', { options })

interface ExtractOptions {
  input_path:      string;
  frame_count:     number;         // 1–MAX_FRAMES
  target_fps:      number;
  fit_mode:        FitMode;        // enum: 'stretch' | 'crop' | 'focus'
  anchor_x?:       number;         // PRE-CLAMPED crop X offset in source pixels (focus mode)
  anchor_y?:       number;         // PRE-CLAMPED crop Y offset in source pixels (focus mode)
  frame_width?:    number;         // source width (needed for focus scale step in Rust)
  frame_height?:   number;         // source height
  reduction_mode?: ReductionMode;  // enum: 'trim_start' | 'trim_end' | 'interpolate'
  duration_secs?:  number;         // required for interpolate fps calculation
}

interface ExtractResult {
  temp_dir:     string;  // path to dir containing frame_0001.png … frame_NNNN.png
  actual_count: number;  // real number of PNGs produced
}
```

**Focus mode anchor math (computed in `ExportView.ts` before calling `extractFrames`):**
```typescript
cropX = Math.max(0, Math.min(anchorX - CELL_SIZE / 2, frameW - CELL_SIZE));
cropY = Math.max(0, Math.min(anchorY - CELL_SIZE / 2, frameH - CELL_SIZE));
// where CELL_SIZE = 128
```

### `assemble_sheet`
```typescript
// Frontend call
assembleSheet(options: AssembleOptions): Promise<AssembleResult>
// invoke<AssembleResult>('assemble_sheet', { options })

interface AssembleOptions {
  temp_dir:     string;
  frame_count:  number;  // use actual_count from ExtractResult
  output_path:  string;  // full OS path from save dialog
}

interface AssembleResult {
  output_path:     string;
  file_size_bytes: number;
  preview_base64:  string; // base64 PNG of the full 1024×1024 sheet (for canvas animation)
}
```

### `cleanup_temp`
```typescript
cleanupTemp(tempDir: string): Promise<void>
// invoke<void>('cleanup_temp', { tempDir })
// Rust: std::fs::remove_dir_all()
```

### `open_file_dialog`
```typescript
openFileDialog(): Promise<string | null>
// Filters: gif, mp4, webm, mov, avi, mkv
// Returns: OS file path or null if cancelled
```

### `save_file_dialog`
```typescript
saveFileDialog(defaultPath: string): Promise<string | null>
// Filter: png
// Default filename: derived from defaultPath's basename
```

```typescript
enum MediaFormat  { GIF = 'gif', VIDEO = 'video' }
enum ReductionMode { TRIM_START = 'trim_start', TRIM_END = 'trim_end', INTERPOLATE = 'interpolate' }

```

### Constants

```typescript

const DEFAULT_FRAME_COUNT = 16;  // Fallback when source is unknown or > MAX_FRAMES
const DEFAULT_FPS        = 24;
const DEFAULT_FIT_MODE   = FitMode.CROP;
```

### AppState interface

```typescript
interface AppState {
  inputPath:     string | null;          // Full OS path to input file
  mediaInfo:     MediaInfo | null;       // Set after analyze_media
  reductionMode: ReductionMode | null;   // null when total_frames ≤ MAX_FRAMES
  fitMode:       FitMode;               // default: FitMode.CROP
  anchorX:       number | null;          // Source-space pixel, for focus mode
  anchorY:       number | null;
  frameCount:    number;                 // 1–MAX_FRAMES, default: DEFAULT_FRAME_COUNT
  fps:           number;                 // default: DEFAULT_FPS
  outputName:    string;                 // Stem of output filename (no extension)
  tempDir:       string | null;          // Set after extract_frames, cleared after cleanup
  previewBase64: string | null;          // Cached first-frame PNG for FitView
}
```

### Helpers exported from `state.ts`

- **`resetState()`** — resets all fields to defaults. Call before navigating back to `DropView`.
- **`buildOutputFilename()`** — returns `<Name>_<N>frames_<FPS>fps.png` from current state. Import this; do not re-implement in views.
- **`defaultFrameCount(totalFrames)`** — returns the appropriate `frameCount` default for a freshly analyzed file.

## 8. View System

### Navigation Flow

```
DropView (step 0)
  → AnalysisView (step 1)
      ├── [total_frames > 64] → ReductionView (step 2) → FitView (step 3)
      └── [total_frames ≤ 64] → FitView (step 3)
  → OptionsView (step 4)
  → ExportView (step 5)
```

### View Interface (`src/views/View.ts`)

```typescript
export interface View {
  mount(container: HTMLElement): void;
  unmount(): void;
}
```

**Rules for every view:**
1. `mount()` builds DOM using `html\`…\`` helper or `document.createElement`.
2. All event listeners **must be named functions** (not inline lambdas) so `removeEventListener` works.
3. Push every `removeEventListener` call to `this.listeners: Array<() => void>`.
4. `unmount()` calls all `this.listeners`. Never skip this.
5. Navigation is done via `import('./SomeView').then(...)` (lazy) + `router.navigate(view, stepIndex)`.
6. Views are lazy-imported to keep the initial bundle small.
7. Use enum values (`FitMode.CROP`, `ReductionMode.INTERPOLATE`) — never raw strings.

### ViewRouter (`src/router.ts`)

```typescript
router.navigate(view: View, step: StepIndex): void
// step: 0–5, updates the 6-segment step indicator bar
```

Step indicator: `.step-seg` elements in `index.html`. States: `active` (current), `done` (past), default (future).

### UI Helpers

- **`html\`...\``** (`src/ui/html.ts`): Tagged template → `HTMLElement` via `<template>`. Returns `firstElementChild`.
- **`ProgressBar`** (`src/ui/progress.ts`): `new ProgressBar()` → `.elements: [track, statusText]`. Call `.setProgress(pct, msg)` and `.reset()`.

## 9. Rust Backend Architecture

### Command Registration (`src-tauri/src/lib.rs`)

All commands registered in `tauri::generate_handler![...]`:
- `analyze_media`
- `extract_frames`
- `extract_preview`
- `assemble_sheet`
- `cleanup_temp`
- `open_file_dialog`
- `save_file_dialog`

Plugins: `tauri_plugin_opener`, `tauri_plugin_shell`, `tauri_plugin_dialog`.

### FFmpeg Filter Logic (`commands/extract.rs → build_vf()`)

| Fit mode | `-vf` filter |
|---|---|
| `stretch` | `scale=128:128` |
| `crop` | `scale='if(gt(a,1),128,-1)':'if(gt(a,1),-1,128)',crop=128:128` |
| `focus` | `crop=128:128:<cx>:<cy>` (with optional upscale if source < 128px) |

**Reduction modes (prepended or combined):**

| Mode | Strategy |
|---|---|
| `trim_start` | `-frames:v N` from beginning |
| `trim_end` | `-sseof -<duration>` seek from end, then `-frames:v N` |
| `interpolate` | `minterpolate=fps=<N/duration>:mi_mode=mci,trim=end_frame=<N>,setpts=PTS-STARTPTS,<scale>` |

**Frame output pattern:** `frame_%04d.png` (1-indexed: `frame_0001.png` → `frame_NNNN.png`).

### Sprite Sheet Assembly (`commands/assemble.rs`)

- Canvas: 1024×1024 RGBA, initialized transparent.
- Iterates frames 1..=frame_count, reads `frame_{i:04}.png`.
- `col = i % 8`, `row = i / 8`, placed at `(col*128, row*128)`.
- If a frame file is missing, stops early (transparent cell).
- If frame dims ≠ 128×128, resizes with Lanczos3.
- Encodes to PNG in memory → base64 → `preview_base64` field.
- Saves to `output_path`.

### Base64 Encoding

Both `extract.rs` and `assemble.rs` implement a custom `encode_base64()` using only `std`. No external crate needed. Do not add a crate for this.

## 11. FFmpeg Binaries (Dynamic Download)

Instead of bundling a massive ~200MB FFmpeg alongside the app, the `.exe` is lightweight (~8MB).
On the **first launch**, the frontend calls `download_dependencies`, which:
1. Downloads a ZIP of the latest FFmpeg build from GitHub using `reqwest`.
2. Extracts `ffmpeg.exe` and `ffprobe.exe` into the OS's local app data directory (`%APPDATA%` / `app_local_data_dir()`).
3. Future runs call `check_dependencies` and skip the download if they exist.

Commands (`extract.rs`, `analyze.rs`) run these downloaded binaries via `shell.command("path/to/exe")` instead of `shell.sidecar()`.

## 14. Known Patterns & Gotchas

### IPC Argument Wrapping

Tauri v2 requires arguments to be wrapped in an object matching the Rust parameter name:

```typescript
// ✅ Correct — Rust fn signature: pub async fn extract_frames(options: ExtractOptions)
invoke('extract_frames', { options })

// ✅ Correct — Rust fn signature: pub async fn cleanup_temp(temp_dir: String)
invoke('cleanup_temp', { tempDir })   // camelCase JS → snake_case Rust auto-converted

// ✅ Correct — single arg
invoke('analyze_media', { path })
```

> Tauri **auto-converts camelCase ↔ snake_case** for IPC arguments.

### Preview Flow

1. `FitView` calls `extractPreview(inputPath)` on demand (only when focus mode is selected).
2. Result stored in `state.previewBase64` — not re-fetched if already cached.
3. `ExportView` plays back the sprite sheet using a `<canvas>` driven by `requestAnimationFrame` reading from `assembleResult.preview_base64` (the full 1024×1024 sheet, animated by slicing 128×128 cells).

### Temp Directory Lifecycle

- Created in `extract_frames` via `TempDir::new()` then `.keep()` (persists past function return).
- Path stored in `state.tempDir`.
- Deleted by `cleanup_temp()` after successful assembly OR on user cancel.
- Call `cleanup_temp` in `ExportView` error paths too.

### Frame Count Defaults in DropView

Use `defaultFrameCount()` from `state.ts` — do not inline this logic:

```typescript
state.frameCount = defaultFrameCount(info.total_frames);
// Returns info.total_frames if 0 < total_frames ≤ MAX_FRAMES, else DEFAULT_FRAME_COUNT.
// When total_frames > MAX_FRAMES: stays at DEFAULT_FRAME_COUNT until ReductionView/OptionsView.
```

### Drag-Drop Events

`DropView` uses Tauri's `getCurrentWebview().onDragDropEvent()` for real OS file paths. HTML5 drag events are kept only for visual highlight (`.drag-over` class). The actual file path comes exclusively from the Tauri event `payload.paths[0]`.

A debounce guard (`lastDropPath`) prevents duplicate `drop` events from firing twice.

## 16. Out of Scope (v1)

Do **not** implement these — they are explicitly excluded:

- Palette/color optimization (no indexed PNG)
- Animated preview playback (static preview only — actually implemented as canvas in ExportView, but no live GIF/video preview)
- Batch processing (one file at a time)
- Cloud upload / VRChat API integration
- Audio extraction
- Custom grid sizes (always 8×8)
- Frame deduplication
- Auto-update mechanism
- Light mode

## 18. Code Style Conventions

All TypeScript source files follow the structural pattern established by `authors.ts`:

### File-level banner

Every file opens with a top-level banner using `=` (not `─`):

```typescript
// =========================================================================================================
// MODULE NAME — purpose summary
// =========================================================================================================
// Longer description of what this module owns, its invariants, and what it does NOT do.
// =========================================================================================================
```

### Section banners within files

Logical groups inside a file use the same `=` style at a reduced width:

```typescript
// =========================================================================
// Section title
// =========================================================================
```

### No hardcoded magic values

- Use constants from `state.ts` (`MAX_FRAMES`, `DEFAULT_FPS`, etc.).
- Use enums (`FitMode`, `ReductionMode`, `MediaFormat`) — never `'crop'` or `'trim_start'` inline.
- Define module-local constants (e.g. `ACCEPTED_EXTENSIONS`, `DROP_DEBOUNCE_MS`) at the top of the file under a `// === Constants ===` banner.

### Named event handlers

Always assign event handlers to named variables before passing them to `addEventListener`:

```typescript
// ✅ Correct — can be removed with removeEventListener
const onBack = async (): Promise<void> => { … };
backBtn.addEventListener('click', onBack);
this.listeners.push(() => backBtn.removeEventListener('click', onBack));

// ❌ Wrong — anonymous lambda cannot be removed
backBtn.addEventListener('click', async () => { … });
```

### Shared logic

If logic is needed in more than one view, it belongs in `state.ts` (state helpers) or a dedicated `src/ui/` module — never duplicated across views.
